# Authentication Fundamentals

## What is Authentication?

Authentication is the process of **verifying that an entity is actually who it claims to be.**

It answers the question:

> **"Can you prove who you are?"**

Identity is just a claim.

Authentication verifies that claim.

---

## Identity vs Authentication

Identity:

```
"I am Anish."
```

Authentication:

```
"Prove it."

↓

Password

↓

Fingerprint

↓

OTP

↓

Verification

↓

Authenticated
```

Identity = Claim

Authentication = Verification

---

## Why Do We Need Authentication?

Without authentication:

- Anyone could access anyone else's account.
- Anyone could transfer money from your bank account.
- Anyone could read your emails.
- Anyone could delete your data.

Authentication exists to verify a user's identity before granting access.

---

## What Problem Does Authentication Solve?

Authentication solves one simple problem:

```
User claims identity

↓

Server verifies claim

↓

Only verified users are trusted
```

Without authentication, servers cannot distinguish between legitimate users and attackers.

## Three Components of Authentication

### 1. Identity Claim

Examples:

- Email
- Username
- Phone Number
- User ID

Example:

```
anish@gmail.com
```

---

### 2. Credential

Proof that supports the identity claim.

Examples:

- Password
- PIN
- OTP
- Fingerprint
- Face Scan
- Security Key

---

### 3. Verification

The server checks whether the provided credential matches the stored information.

```
Claim

↓

Credential

↓

Verification

↓

Authenticated
```

# Authentication Factors

Authentication factors are different categories of evidence used to verify identity.

## 1. Something You Know

Knowledge-based authentication.

Examples:

- Password
- PIN
- Passphrase
- Security Question

---

## 2. Something You Have

Possession-based authentication.

Examples:

- Mobile Phone
- OTP Device
- Authenticator App
- Hardware Security Key
- Smart Card

---

## 3. Something You Are

Biometric authentication.

Examples:

- Fingerprint
- Face Recognition
- Retina Scan
- Voice Recognition
- Palm Scan

---

# Multi-Factor Authentication (MFA)

MFA combines two or more authentication factors.

Example:

```
Password

+

OTP

↓

Authenticated
```

Another example:

```
Password

+

Fingerprint

↓

Authenticated
```

This provides stronger security than using only one factor.

---

# Authentication vs Authorization

Authentication:

```
Who are you?
```

Authorization:

```
What are you allowed to do?
```

Example:

```
User logs in

↓

Authenticated

↓

Role = Student

↓

Can View Notes

↓

Cannot Delete Database
```

# Common Misconceptions

❌ Identity = Authentication

❌ Password = Authentication

❌ JWT = Authentication

❌ Cookie = Authentication

❌ Session = Authentication

✅ Password is a credential.

✅ JWT is a token used after authentication.

✅ Session maintains authenticated state.

✅ Cookie stores/transports data.

✅ Authentication verifies identity.

---

# Key Takeaways

- Authentication verifies identity.
- Identity is only a claim.
- Authentication requires proof.
- Authentication consists of claim, credential, and verification.
- Authentication factors include knowledge, possession, and biometrics.
- MFA combines multiple authentication factors.
- Authentication happens before authorization.
- Sessions and JWTs are mechanisms used after successful authentication.

---

# One-Line Summary

Authentication is the process of verifying that an entity is actually who it claims to be before granting access to protected resources.
