# Identity Fundamentals - Detailed Revision Notes

# 1. What is Identity?

Identity is the collection of attributes that uniquely describe or distinguish an entity.

Simply put:

> Identity answers the question:
>
> **"Who are you?"**

Identity can belong to:

- Person
- Computer
- Server
- Mobile Device
- API
- Company
- IoT Device
- Service

Identity is not limited to humans.

---

# 2. Identity vs Authentication

Identity:

Claims who you are.

Example:

```text
"I am Anish."
```

Authentication:

Verifies that claim.

Example:

```text
Person:
"I am Anish."

↓

Show ID Card

↓

Verified

↓

Authenticated
```

Easy Memory:

```text
Identity

↓

Claim

↓

Authentication

↓

Verification
```

Identity is a claim.

Authentication is proof.

---

# 3. Digital Identity

Digital Identity is the collection of digital information that uniquely identifies a user or system.

Examples:

```text
User ID

Username

Email

Phone Number

Employee ID

Student Roll Number

Passport Number

Google Account ID

GitHub Username
```

Computers cannot recognize:

- Face
- Voice
- Personality

They identify users using digital attributes.

# 5. Unique Identifier

A Unique Identifier is a value that uniquely identifies exactly one entity within a system.

No duplicates are allowed.

Purpose:

- Remove ambiguity
- Ensure uniqueness
- Enable reliable lookup

---

# 6. Why Unique Identifiers Exist

Without IDs:

```text
Rahul

Rahul

Rahul

Rahul
```

Delete Rahul.

Impossible to know which one.

With IDs:

```text
ID     Name

1      Rahul

2      Rahul

3      Rahul

4      Rahul
```

Delete:

```text
User 3
```

No ambiguity.

# 8. Internal vs External Identity

External Identity:

Can change.

Examples:

- Name
- Username
- Email
- Phone Number

Internal Identity:

Usually never changes.

Examples:

- Database ID
- UUID
- Employee ID

Example:

```text
ID: 42

Username: anish

Email: anish@gmail.com

Phone: 9876543210
```

Even if username changes:

```text
anish

↓

anish_007
```

ID remains:

```text
42
```

Backend systems usually rely on internal IDs.

---

# 9. Identity in Backend Systems

Typical User Table:

```text
id    username      email

1     anish         anish@gmail.com

2     rahul         rahul@gmail.com
```

Authentication Flow:

```text
User enters email

↓

Server finds matching user

↓

Gets internal User ID

↓

Verifies password

↓

Authentication successful

↓

Future operations use internal ID
```

# Key Takeaways

- Identity answers "Who are you?"
- Identity is only a claim.
- Authentication verifies identity.
- Computers use digital identity instead of physical characteristics.
- Digital identity consists of digital attributes.
- Unique identifiers remove ambiguity.
- Backend systems usually rely on stable internal IDs.
- Internal IDs should rarely change.
- Identity can belong to people, devices, or services.

---

# One-Line Summary

Identity is the collection of attributes that uniquely identify an entity, while authentication is the process of verifying that claimed identity.
