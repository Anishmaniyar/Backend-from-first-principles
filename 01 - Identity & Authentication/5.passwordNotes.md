# Password Fundamentals

## What is a Password?

A password is a **secret known by the user and the authentication system** that is used to verify identity.

It belongs to the authentication factor:

> **Something You Know**

A password is **not authentication itself**.

It is a credential used during the authentication process.

---

# Why Do Passwords Exist?

Passwords exist to solve one fundamental problem:

> **How can a user prove they are really who they claim to be?**

Identity alone is only a claim.

Example:

```
"I am Anish."
```

Anyone can make that claim.

The password provides proof.

```
Identity Claim

↓

"I am Anish"

↓

Password

↓

Verification

↓

Authenticated
```

Think of a password as a digital key.

---

# Why Isn't Identity Alone Enough?

Suppose a login page only asks for:

```
Email:

anish@gmail.com
```

If no password existed:

```
Enter Email

↓

Login Successful
```

Anyone who knows your email could access your account.

Passwords prevent this by requiring secret knowledge.

---

# Password-Based Authentication

Password-based authentication is an authentication mechanism where a user proves their identity using a password.

Typical Flow:

```
User

↓

Email

+

Password

↓

Server

↓

Find User

↓

Verify Password

↓

Authenticated

↓

Create Session / JWT

↓

Future Requests
```

Authentication occurs when the server verifies the password.

Sessions and JWTs come afterwards.

---

# Problems With Passwords

Passwords are simple and widely used but have many security and usability problems.

---

## 1. Weak Passwords

Examples:

```
123456

password

admin

qwerty

abc123
```

Easy to guess.

---

## 2. Password Reuse

Using the same password across multiple websites.

Example:

```
Netflix

GitHub

Instagram

Amazon

Bank
```

One breach can compromise every account.

---

## 3. Password Theft

Passwords can be stolen through:

- Phishing
- Malware
- Database leaks
- Keyloggers
- Fake websites

---

## 4. Human Memory

Strong passwords are difficult to remember.

Users often:

- Forget passwords
- Reuse passwords
- Write passwords down

---

## 5. Server Storage Problem

Servers should NEVER store:

```
password123
```

Instead:

```
Password

↓

Hash

↓

Store Hash
```

Password hashing exists to solve this problem.

---

# Strong vs Weak Passwords

## Weak Password

Examples:

```
123456

password

admin

anish123

welcome
```

Characteristics:

- Short
- Predictable
- Common
- Easy to guess

---

## Strong Password

Examples:

```
PurpleTrainCoffee91!

river-lamp-orange-mountain

F7#xL2@kP9!qM8
```

Characteristics:

- Long
- Unique
- Random
- Difficult to guess
- Not reused
- Not based on personal information

---

# Characteristics of a Good Password

A strong password should be:

- Long
- Unique
- Random
- Difficult to predict
- Used for only one account
- Not contain personal information

Avoid:

- Name
- Birthday
- Phone Number
- Pet Name
- Dictionary words
- Common patterns

---

# Password Entropy (High-Level)

Entropy represents how difficult a password is to guess.

Simple idea:

```
More Possible Combinations

↓

Harder to Guess

↓

Higher Security
```

Higher randomness generally increases entropy.

---

# Common Password Attacks

## 1. Brute Force Attack

Attacker tries every possible password combination.

Example:

```
a

aa

aaa

aab

aac

...

password123
```

Very slow but eventually succeeds if enough time is available.

---

## 2. Dictionary Attack

Attacker tries commonly used passwords instead of every possible combination.

Examples:

```
password

123456

qwerty

welcome

admin

football
```

Much faster than brute force.

---

## 3. Credential Stuffing

Uses leaked credentials from one website to access accounts on another website.

Example:

```
Netflix Leak

↓

Email + Password

↓

Try Same Credentials

↓

Gmail

GitHub

Amazon

Bank
```

Works because users reuse passwords.

---

## 4. Phishing

Attacker creates a fake login page.

User voluntarily enters:

```
Email

Password
```

Attacker steals credentials.

No password cracking required.

---

## 5. Keylogger

Malicious software records keyboard input.

Example:

```
H

e

l

l

o

1

2

3
```

Attacker obtains the password.

---

## 6. Database Leak

Server stores passwords in plaintext.

Database leaks.

Every password becomes visible.

Example:

```
Database

↓

password123

admin123

hello123
```

This is why password hashing is critical.

---

# Password Managers

Password managers help users:

- Generate strong passwords
- Store passwords securely
- Autofill credentials
- Avoid password reuse

Example:

```
Random Password

↓

Password Manager

↓

Stored Securely

↓

Auto-filled Later
```

# Why Modern Systems Are Moving Beyond Passwords

Passwords have many limitations:

- Forgotten
- Reused
- Guessed
- Phished
- Stolen

Modern alternatives include:

- Passkeys
- Biometrics
- Hardware Security Keys
- Multi-Factor Authentication (MFA)
- WebAuthn

However, passwords are still one of the most common authentication methods.

# Common Misconceptions

## ❌ Password = Authentication

False.

Password is a credential.

Authentication is the verification process.

---

## ❌ Longer Always Means Stronger

False.

Predictable passwords can still be weak.

---

## ❌ Complex Symbols Automatically Make Passwords Secure

False.

Length, randomness, and uniqueness are more important.

---

## ❌ Password Reuse Is Safe

False.

One leaked password can compromise multiple accounts.

---

## ❌ Servers Should Store Passwords

False.

Servers should store password hashes, never plaintext passwords.

---

# Key Takeaways

- Passwords exist to prove identity.
- Passwords are credentials, not authentication.
- Identity alone is not sufficient.
- Password-based authentication is widely used.
- Weak passwords are easy to guess.
- Password reuse is dangerous.
- Password theft is a major security risk.
- Servers should never store plaintext passwords.
- Strong passwords should be long, unique, and random.
- Password managers encourage better security practices.
