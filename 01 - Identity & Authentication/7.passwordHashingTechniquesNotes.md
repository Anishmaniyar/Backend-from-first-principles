# Password Hashing Algorithms

## Why Do We Need Password Hashing Algorithms?

Simply hashing passwords with fast cryptographic hash functions like SHA-256 is not sufficient.

Modern attackers have:

- Powerful CPUs
- GPUs
- Cloud computing
- Specialized hardware
- Massive parallel processing

Fast hashing makes brute-force attacks practical.

Password hashing algorithms are intentionally designed to make password guessing slow and expensive.

---

# Why Not SHA-256?

SHA-256 is an excellent cryptographic hash function.

However, it was designed to be:

- Fast
- Efficient
- Deterministic

These properties are desirable for:

- File integrity
- Digital signatures
- Blockchain
- Checksums

But they are undesirable for password storage.

Example:

```
Password

↓

SHA-256

↓

Very Fast

↓

Attacker Can Try Millions/Billions of Guesses
```

Fast hashing makes brute-force attacks much easier.

---

# Password Hashing vs General Hashing

General cryptographic hashing aims for:

- Speed
- Integrity
- Verification

Password hashing aims for:

- Slow computation
- High attack cost
- Secure password storage

For passwords:

```
Slower

=

More Secure
```

---

# bcrypt

bcrypt is a password hashing algorithm specifically designed for secure password storage.

It intentionally slows down password hashing to make brute-force attacks expensive.

bcrypt automatically:

- Generates a random salt
- Combines password and salt
- Computes the hash
- Stores salt information inside the final hash

Developers typically only store:

```
password_hash
```

No separate salt column is usually required.

---

# bcrypt Signup Flow

```
Password

↓

Generate Salt

↓

Combine

↓

bcrypt

↓

Store Hash
```

---

# bcrypt Login Flow

```
Password

↓

bcrypt.compare()

↓

Extract Salt

↓

Hash Again

↓

Compare

↓

Authenticated
```

The original password is never stored.

---

# Cost Factor

bcrypt provides a configurable cost factor.

The cost factor determines how computationally expensive hashing should be.

Lower cost:

```
Fast
```

Higher cost:

```
Slower

↓

Harder to Brute Force
```

As hardware becomes faster, applications can increase the cost factor to maintain security.

---

# Why Slow Hashing Helps

Attacker:

```
Guess Password

↓

Hash

↓

Compare

↓

Repeat
```

If each guess is expensive:

- Fewer guesses per second
- Higher attack cost
- Better password protection

---

# Argon2

Argon2 is a modern password hashing algorithm designed specifically for secure password storage.

It won the Password Hashing Competition (PHC) in 2015.

Argon2 provides:

- Built-in salt generation
- Configurable computation cost
- Memory hardness
- Resistance to GPU attacks
- Modern security design

It is widely considered one of the best choices for password hashing today.

---

# Memory Hardness

Memory hardness means the algorithm intentionally requires significant memory (RAM) in addition to CPU computation.

Instead of requiring only:

```
CPU
```

Argon2 requires:

```
CPU

+

Memory
```

This makes massive parallel attacks much more expensive.

---

# Why Memory Hardness Matters

Modern GPUs can perform enormous numbers of computations simultaneously.

Argon2 forces attackers to also allocate significant memory for every password guess.

Result:

- Higher hardware cost
- Less efficient GPU attacks
- Better resistance to large-scale brute-force attacks

# Common Misconceptions

## ❌ SHA-256 is insecure

False.

SHA-256 is extremely secure.

It is simply too fast for password storage.

---

## ❌ bcrypt encrypts passwords

False.

bcrypt hashes passwords.

No decryption is possible.

---

## ❌ Argon2 is completely different from bcrypt

False.

Both solve the same problem.

Argon2 improves resistance against modern hardware attacks.

---

## ❌ Salt must be managed manually

Usually false.

bcrypt and Argon2 automatically generate and manage salts internally.

---

## ❌ Password hashing makes passwords impossible to crack

False.

Weak passwords can still be guessed.

Password hashing simply makes every guess significantly more expensive.

---

# Key Takeaways

- SHA-256 is designed for fast cryptographic hashing.
- Fast hashing is undesirable for password storage.
- bcrypt intentionally slows hashing.
- bcrypt automatically generates salts.
- Cost factor controls hashing expense.
- Argon2 is a modern password hashing algorithm.
- Argon2 introduces memory hardness.
- Memory hardness increases the cost of GPU attacks.
- bcrypt and Argon2 are designed specifically for secure password storage.
