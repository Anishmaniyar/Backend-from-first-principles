# HTTP Headers - Quick Revision Notes

# 1. What are HTTP Headers?

HTTP Headers are **metadata (information about the request or response)** sent between the client and server.

They provide context but are **not the actual data**.

Think:

```text
Package

-------------------
Label (Headers)
-------------------

Actual Product (Body)
-------------------
```

Headers describe the data.

The Body contains the data.

---

# 2. Why Do Headers Exist?

Headers allow the client and server to exchange additional information such as:

- Authentication
- Data format
- Browser information
- Cookies
- Language preferences
- Cache information
- Content type

Without headers, the server would have very little context about the request.

---

# 3. Header Format

Every header follows:

```http
Header-Name: Value
```

Example:

```http
Content-Type: application/json

Authorization: Bearer abc123

Host: api.example.com
```

---

# 4. Request vs Response Headers

Request Headers

Client → Server

Example:

```http
GET /profile

Host: api.example.com

Authorization: Bearer xyz

Accept: application/json
```

---

Response Headers

Server → Client

Example:

```http
200 OK

Content-Type: application/json

Set-Cookie: session=abc123
```

---

# 5. Most Important Request Headers

## A. Host

Purpose:
Identifies which website/server the client wants.

Example:

```http
Host: github.com
```

Why needed:
One physical server can host multiple websites.

Example:

```text
Same Server

├── github.com

├── api.github.com

└── docs.github.com
```

Host tells the server which one to serve.

---

## B. Content-Type

Purpose:
Tells the receiver what format the body is in.

Example:

```http
Content-Type: application/json
```

Body:

```json
{
  "email": "anish@gmail.com"
}
```

Common values:

```text
application/json

text/html

text/plain

multipart/form-data

application/xml
```

Used heavily in:

- APIs
- Login
- Signup
- File Uploads

---

## C. Accept

Purpose:
Tells the server what response format the client prefers.

Example:

```http
Accept: application/json
```

Meaning:

"I would like JSON in the response."

Examples:

```http
Accept: application/json

Accept: text/html

Accept: image/png
```

---

## D. Authorization

Purpose:
Sends authentication credentials to the server.

Example:

```http
Authorization: Bearer eyJhbGc...
```

Used for:

- JWT
- OAuth
- API Tokens
- Bearer Tokens

Typical Flow:

```text
Login

↓

Receive Token

↓

Store Token

↓

Send in Authorization Header

↓

Server Verifies

↓

Access Granted
```

---

## E. Cookie

Purpose:
Sends previously stored cookies from browser to server.

Example:

```http
Cookie: session=abc123
```

Browser automatically includes cookies in future requests.

Used for:

- Sessions
- Authentication
- User Preferences

---

## F. User-Agent

Purpose:
Identifies the client making the request.

Example:

```http
User-Agent: Chrome
```

Can contain:

- Browser
- Version
- Operating System
- Device Information

Used for:

- Analytics
- Logging
- Compatibility
- Debugging

---

# 6. Important Response Headers

## A. Content-Type

Specifies response format.

Example:

```http
Content-Type: application/json
```

Response:

```json
{
  "success": true
}
```

---

## B. Set-Cookie

Purpose:
Server tells browser to store a cookie.

Example:

```http
Set-Cookie: session=abc123
```

Browser stores it automatically.

Future requests:

```http
Cookie: session=abc123
```

---

## C. Cache-Control

Controls browser/server caching behavior.

Example:

```http
Cache-Control: no-cache
```

Examples:

- no-cache
- no-store
- max-age=3600

---

## D. Location

Used during redirects.

Example:

```http
Location: /login
```

Commonly used with:

- Redirects
- OAuth
- Login Flows

---

# 7. Cookie vs Set-Cookie

Server:

```http
Set-Cookie: session=abc123
```

↓

Browser stores cookie

↓

Future Request:

```http
Cookie: session=abc123
```

Memory Trick:

Server SETS

Browser SENDS

---

# 8. Header vs Body

Headers

- Metadata
- Extra information
- Authentication
- Content Type
- Browser Info

Body

- Actual Data
- JSON
- Images
- Files
- Email
- Password

Example:

```http
Headers

Content-Type: application/json

Authorization: Bearer xyz

----------------------

Body

{
  "email":"anish@gmail.com",
  "password":"123"
}
```

---

# 9. Authentication Example

Request:

```http
POST /login

Headers

Content-Type: application/json

-------------------

Body

{
  "email":"anish@gmail.com",
  "password":"123456"
}
```

Server:

Verifies credentials

↓

Returns:

```http
200 OK

Headers

Set-Cookie: session=abc123

--------------------

Body

{
  "success": true
}
```

Next Request:

```http
GET /profile

Cookie: session=abc123
```

Server identifies the user.

---

# 10. Key Takeaways

- Headers contain metadata.
- Body contains actual data.
- Content-Type describes body format.
- Accept tells preferred response format.
- Authorization sends authentication credentials.
- Cookie sends stored cookies to server.
- Set-Cookie stores cookies in browser.
- Host identifies the destination website.
- User-Agent identifies the client.
- Cache-Control manages caching.
- Location is used for redirects.

---

# One-Line Summary

HTTP Headers are metadata exchanged between client and server that describe how the request or response should be processed, while the Body contains the actual data.
