# HTTP Methods - Quick Revision Notes

## 1. What are HTTP Methods?

HTTP Methods define **what action a client wants the server to perform** on a resource.

Think of them as **verbs (actions)** in a sentence.

```text
GET /users

GET    → Action (Verb)
users  → Resource (Noun)
```

---

# 2. Why Do HTTP Methods Exist?

Without methods:

```text
/users
```

The server doesn't know what you want to do.

- Read users?
- Create user?
- Update user?
- Delete user?

HTTP methods remove this ambiguity by clearly specifying the intended action.

---

# 3. Core Mental Model

```text
HTTP Method = Action (Verb)

Resource = Thing (Noun)
```

Examples:

```text
GET    /users
POST   /users
PATCH  /users/1
DELETE /users/1
```

---

# 4. GET

## Purpose

Retrieve (read) data.

Think:

> "Give me this resource."

Examples:

```http
GET /users

GET /profile

GET /posts
```

Characteristics:

- Fetches data
- Should NOT modify data
- Safe to call multiple times

Real Examples:

- Open YouTube homepage
- View Instagram feed
- View user profile

---

# 5. POST

## Purpose

Create a new resource.

Think:

> "Create this."

Examples:

```http
POST /users

POST /posts

POST /signup
```

Characteristics:

- Creates new data
- Usually includes a request body
- Commonly used for forms and submissions

Real Examples:

- Signup
- Login
- Create Tweet
- Upload File

---

# 6. PUT

## Purpose

Replace an existing resource completely.

Think:

> "Replace the entire object."

Example:

Before:

```json
{
  "name": "Anish",
  "age": 20,
  "city": "Pune"
}
```

PUT:

```json
{
  "name": "Rahul",
  "age": 25,
  "city": "Mumbai"
}
```

After:

```json
{
  "name": "Rahul",
  "age": 25,
  "city": "Mumbai"
}
```

Characteristics:

- Full replacement
- Entire resource is updated

---

# 7. PATCH

## Purpose

Update only specific fields of an existing resource.

Think:

> "Change only this part."

Example:

Before:

```json
{
  "name": "Anish",
  "age": 20,
  "city": "Pune"
}
```

PATCH:

```json
{
  "age": 21
}
```

After:

```json
{
  "name": "Anish",
  "age": 21,
  "city": "Pune"
}
```

Characteristics:

- Partial update
- Only changed fields are sent

---

# 8. DELETE

## Purpose

Remove a resource.

Think:

> "Delete this."

Examples:

```http
DELETE /users/15

DELETE /posts/3

DELETE /comments/8
```

Characteristics:

- Removes existing data

---

# 9. PUT vs PATCH

PUT:

- Replaces the entire resource
- Sends complete object

PATCH:

- Updates only required fields
- Sends partial object

Example:

PUT:

```json
{
  "name": "Anish",
  "age": 21,
  "city": "Pune"
}
```

PATCH:

```json
{
  "age": 21
}
```

---

# 10. CRUD Mapping

| CRUD Operation   | HTTP Method |
| ---------------- | ----------- |
| Create           | POST        |
| Read             | GET         |
| Update (Full)    | PUT         |
| Update (Partial) | PATCH       |
| Delete           | DELETE      |

Easy Memory Trick:

```text
Create  → POST

Read    → GET

Update  → PUT / PATCH

Delete  → DELETE
```

# 12. Restaurant Analogy

GET
→ Show me the menu.

POST
→ Place a new order.

PUT
→ Replace my entire order.

PATCH
→ Change only the drink.

DELETE
→ Cancel my order.

---

# Key Takeaways

- HTTP methods tell the server what action to perform.
- They remove ambiguity in client-server communication.
- GET is used to read data.
- POST is used to create data.
- PUT replaces an entire resource.
- PATCH updates only part of a resource.
- DELETE removes a resource.
- Think of methods as verbs and resources as nouns.

---

# One-Line Summary

> HTTP methods are standardized actions that tell the server what operation (read, create, update, or delete) should be performed on a resource.
