# HTTP Status Codes - Detailed Revision Notes

# 1. What are HTTP Status Codes?

HTTP Status Codes are **standardized numerical codes sent by the server to indicate the result of processing a request.**

They tell the client:

- Was the request successful?
- Was there an error?
- Should it redirect?
- Was the client wrong?
- Did the server fail?

Think of them as:

> **The server's final verdict on your request.**

---

# 2. Why Do Status Codes Exist?

Without status codes, the client would not know whether:

- Request succeeded
- Resource exists
- Authentication failed
- Permission denied
- Server crashed

Status codes provide a universal language between clients and servers.

# 4. Status Code Categories

## 1xx → Informational

Meaning:

- Request received
- Processing continues

Examples:

- 100 Continue
- 101 Switching Protocols

Usually not important for beginners.

---

## 2xx → Success

Meaning:

- Request processed successfully.

Examples:

- 200 OK
- 201 Created
- 204 No Content

---

## 3xx → Redirection

Meaning:

- Resource moved
- Client should go somewhere else

Examples:

- 301 Moved Permanently
- 302 Found

---

## 4xx → Client Error

Meaning:

- Client made a mistake

Examples:

- 400 Bad Request
- 401 Unauthorized
- 403 Forbidden
- 404 Not Found
- 409 Conflict
- 422 Unprocessable Entity

---

## 5xx → Server Error

Meaning:

- Server failed while processing request

Examples:

- 500 Internal Server Error

---

# 5. Most Important Status Codes

## 200 OK

Meaning:
Request completed successfully.

Typical Usage:

- Get Profile
- Fetch Users
- Login Success
- Fetch Products

Example:

```http
GET /profile

↓

200 OK
```

Memory:

> Everything worked.

---

## 201 Created

Meaning:
A new resource was successfully created.

Typical Usage:

- Signup
- Create User
- Create Post
- Upload Resource

Example:

```http
POST /users

↓

201 Created
```

Memory:

> Successfully created something new.

---

## 204 No Content

Meaning:
Request succeeded but there is nothing to return.

Typical Usage:

- Delete operations
- Empty successful responses

Example:

```http
DELETE /posts/10

↓

204 No Content
```

Memory:

> Success, but no response body.

---

# 6. Redirect Status Codes

## 301 Moved Permanently

Meaning:
Resource has permanently moved.

Example:

```
oldsite.com

↓

newsite.com
```

Browser can permanently update the URL.

---

## 302 Found

Meaning:
Temporary redirect.

Typical Usage:

- Login redirects
- OAuth redirects
- Temporary routing

Example:

```
/profile

↓

Not Logged In

↓

302

↓

/login
```

Memory:

> Go somewhere else temporarily.

---

# 7. Client Error Status Codes

## 400 Bad Request

Meaning:
The client sent an invalid request.

Examples:

- Invalid JSON
- Missing required fields
- Incorrect request format

Memory:

> Bad request from client.

---

## 401 Unauthorized

Meaning:
Client is NOT authenticated.

Server does not know who the user is.

Examples:

- No JWT
- No Session
- Invalid Token
- Missing Authentication

Memory:

> Login required.

Think:

```
Who are you?

I don't know.

Login first.
```

---

## 403 Forbidden

Meaning:
Client is authenticated but does NOT have permission.

Examples:

- Student trying admin route
- User deleting another user's data
- Missing required role

Memory:

> I know who you are, but you cannot do this.

Think:

```
Identity ✓

Permission ✗
```

---

## 404 Not Found

Meaning:
Requested resource does not exist.

Examples:

```
GET /users/99999

↓

404
```

Memory:

> Resource not found.

---

## 409 Conflict

Meaning:
Request conflicts with existing data.

Examples:

- Email already registered
- Username already exists
- Duplicate unique value

Memory:

> Data conflict.

---

## 422 Unprocessable Entity

Meaning:
Request format is valid but validation fails.

Examples:

Valid JSON:

```json
{
  "age": -5
}
```

But:

Age cannot be negative.

Memory:

> Format correct, data invalid.

---

# 8. Server Error Status Codes

## 500 Internal Server Error

Meaning:
Unexpected server-side failure.

Examples:

- Database crash
- Unhandled exception
- Programming bug
- Internal failure

Memory:

> Server problem, not client problem.

---

# 9. 400 vs 500

400:

```
Client made mistake.
```

Examples:

- Wrong data
- Missing field
- Invalid request

---

500:

```
Server made mistake.
```

Examples:

- Crash
- Exception
- Database failure

Easy Memory:

```
400 → Your fault

500 → My fault
```

---

# 10. 401 vs 403

401:

```
Who are you?

↓

Unknown

↓

Login Required
```

Authentication Problem.

---

403:

```
Who are you?

↓

Known

↓

Permission Denied
```

Authorization Problem.

Easy Memory:

```
401

Identity Missing

--------------

403

Permission Missing
```

---

# 11. 200 vs 201

200:

Request succeeded.

Example:

```
GET /users
```

Nothing new created.

---

201:

Something new was created.

Example:

```
POST /signup
```

New user created.

Easy Memory:

```
200

Everything worked

--------------

201

Created something new
```

---

# 12. 204 vs 200

200:

Usually returns data.

Example:

```json
{
  "success": true
}
```

---

204:

Returns NO body.

Example:

```
DELETE /post/5

↓

204
```

No response content needed.

# One-Line Summary

HTTP Status Codes are standardized response codes that tell the client whether a request succeeded, failed, was redirected, or encountered a client-side or server-side error.
