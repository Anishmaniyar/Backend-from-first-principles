# URL Anatomy - Quick Revision Notes

## 1. What is a URL?

**URL (Uniform Resource Locator)** is the complete address of a resource on the internet.

Example:

```text
https://example.com/users/15?page=2
```

Think of it as the address of a house.

---

# 2. Why Do We Need URLs?

URLs help clients locate and access specific resources on a server.

Without URLs, browsers wouldn't know:

- Where to connect
- Which resource to request

---

# 3. URL Structure

```text
https://api.example.com:443/users/15?page=2&sort=name#comments
```

Consists of:

- Protocol
- Domain
- Port
- Path
- Query Parameters
- Fragment

---

# 4. Protocol

Defines how communication happens.

Examples:

- http
- https

Example:

```text
https://example.com
```

Protocol:

```text
https
```

---

# 5. Domain

Human-readable name of the server.

Example:

```text
google.com
github.com
chatgpt.com
```

DNS converts domain names into IP addresses.

---

# 6. Port

Identifies which service on the server should handle the request.

Examples:

```text
localhost:3000
localhost:5000
```

Common Ports:

- 80 → HTTP
- 443 → HTTPS
- 3000 → Next.js Dev
- 5000 → Express (common)
- 5432 → PostgreSQL
- 6379 → Redis

---

# 7. Path

Identifies the resource.

Examples:

```text
/users

/posts

/profile

/users/15
```

Example:

```text
GET /users/15
```

means:

Fetch user with ID 15.

---

# 8. Path Parameters

Represent the identity of a resource.

Examples:

```text
/users/15

/products/10

/posts/25
```

Here:

15, 10 and 25 are path parameters.

---

# 9. Query Parameters

Represent options or filters while retrieving data.

Examples:

```text
?page=2

?sort=name

?limit=20

?category=electronics
```

Multiple query parameters:

```text
?page=2&limit=20&sort=name
```

Starts with:

```text
?
```

Additional parameters use:

```text
&
```

---

# 10. Path vs Query

Path:

- Identifies WHAT resource
- Represents identity

Examples:

```text
/users/15

/products/5
```

Query:

- Modifies HOW data is retrieved
- Represents options, filters and search criteria

Examples:

```text
/users?page=2

/products?sort=price

/search?q=iphone
```

Memory Trick:

Path → WHO / WHAT

Query → HOW

---

# 11. Fragment

Example:

```text
https://docs.example.com/page#installation
```

Fragment:

```text
#installation
```

Used by the browser to jump to a specific section on the page.

## Fragments are generally NOT sent to the server.

# Key Takeaways

- URL is the address of a resource.
- Protocol defines communication rules.
- Domain identifies the server.
- Port identifies the service.
- Path identifies the resource.
- Query parameters modify retrieval behavior.
- Fragment identifies a section within a page.
- Path = Identity
- Query = Options
